## 요구사항
어떤 요구사항에 대한 코드인가요?

ExecutorUnit이 1일차 PDF 스펙의 나머지 Statement 4종(ExpressionStmt,
PrintStmt, IfStmt, ForStmt)을 실행할 수 있어야 한다는 요구사항에 대한
코드입니다. 그와 함께, Environment/Interpreter가 raw pointer에서
shared_ptr<Environment> 소유권 모델로 바뀌면서 EnvironmentTest.cpp에
남아있던 빌드 오류도 같이 정리했습니다.

- ExpressionStmt: 표현식을 평가만 하고 결과는 버린다 (`a + b;`)
- PrintStmt: 표현식을 평가해 stringify한 값을 표준출력에 출력한다 (`print 3 + 2;`)
- IfStmt: 조건이 truthy면 then, 아니면 else 분기를 실행한다
- ForStmt: init을 1회 실행 후 condition이 truthy인 동안 body/increment를 반복한다

변경 범위는 ExecutorUnit 폴더로 한정했습니다(다른 유닛 파일은 건드리지 않음).


## 주요 변경점
어떤 변경점이 있나요?

- `ExecutorUnit/EnvironmentTest.cpp`
  - `NestedEnvironmentTestFixture`가 스택 객체 주소(`Environment inner{ &outer };`)를
    `shared_ptr` 생성자에 넘기던 방식이라 컴파일이 되지 않던 것을
    `make_shared<Environment>` 기반으로 수정
- `ExecutorUnit/Interpreter.h`, `Interpreter.cpp`
  - `execute()`의 `dynamic_cast` 분기에 `ExpressionStmt`/`PrintStmt`/`IfStmt`/`ForStmt`를 추가
  - `executeExpressionStmt`: `evaluate()` 후 결과 폐기
  - `executePrintStmt`: `evaluate()` 후 `stringify()`하여 `std::cout`에 출력
  - `executeIfStmt`: 조건 평가 후 then/else 분기 실행. `getThenBranch()`/`getElseBranch()`가
    `const Statement*`를 반환하므로 `CheckerUnit/Checker.cpp`가 이미 쓰던 것과 동일하게
    `const_cast`로 넘김
  - `executeForStmt`: for 전용 자식 `Environment`를 만들어 init 1회 실행 후,
    condition이 없거나 truthy인 동안 body 실행 + increment 평가를 반복
    (`executeBlockStmt`와 동일하게 종료 시 이전 Environment로 복원)
- `ExecutorUnit/InterpreterTest.cpp`
  - 위 4개 Statement에 대한 TDD 테스트 추가 (`[UnitTest]` → `[FeatureImplement]` 커밋 쌍)
  - `PrintStmt`는 `std::cout.rdbuf()`를 임시 교체해 출력 결과를 캡처해서 검증


## 배움 포인트 (선택)

- 이 브랜치에는 아직 `VariableExpr`/`AssignExpr`/`BinaryExpr` 평가가 없어서(다른
  브랜치에서 진행 중), `ForStmt`의 다회 반복(변수 증가 조건)은 검증할 방법이 없었습니다.
  그래서 이번 테스트는 "condition이 처음부터 falsy라 body가 실행되지 않는 경우"만
  다루고, 실제 반복 동작 검증은 해당 Expression들이 들어온 뒤 후속 커밋으로 미뤘습니다.
- `IfStmt`/`ForStmt`의 자식 노드 getter가 `const Statement*`를 반환하는데
  `execute()`는 `Statement*`를 요구해서, 이미 `Checker::checkIfStmt`가 쓰던
  `const_cast` 패턴을 그대로 따랐습니다(새로운 방식을 만들지 않음).


## 테스트 결과
어떤 테스트를 수행했고 그 결과는 어떤가요?

`InterpreterTest.cpp`에 추가한 테스트가 모두 통과합니다.
- ExpressionStmtEvaluatesInnerExpressionWithoutThrowing / PropagatesEvaluationErrors
- PrintStmtWritesStringifiedValueToStdout
- IfStmtExecutesThenBranchWhenConditionIsTruthy / ExecutesElseBranchWhenConditionIsFalsy / DoesNothingWhenConditionIsFalsyAndNoElseBranch
- ForStmtDoesNotExecuteBodyWhenConditionIsInitiallyFalsy

`EnvironmentTest.cpp`의 `NestedEnvironmentTestFixture` 전체(엔클로징 체인 조회/대입/섀도잉/오류)도 함께 통과합니다.

(참고: 로컬에 MSBuild가 없어 Visual Studio에서의 실제 빌드는 리뷰어분께서 한 번 더 확인해 주시면 감사하겠습니다.)
